# 安全策略

## 为什么安全如此重要

本项目涉及的固件刷写操作直接影响设备的底层系统。内核镜像、设备树、bootloader 配置和 rootfs 中任何一个环节存在安全漏洞，都可能导致：

- 设备变砖（无法恢复的硬件状态）
- 敏感数据泄露（分区表、凭据、密钥）
- 持久化恶意代码植入（内核模块、initramfs、固件）

因此，我们对待安全漏洞的重视程度高于普通软件项目。

## 报告漏洞

如果您发现安全相关的漏洞，**请不要**通过公开的 GitHub Issues 提交，以免在修复前被恶意利用。

### 联系方式

| 渠道 | 说明 |
|------|------|
| **GitHub Issues（私密）** | 如果漏洞属于公开可讨论的范围，可先通过 Issues 联系我们，并说明涉及安全 |
| **Telegram** | 加入我们的社区群：[https://t.me/Pad_6S_Pro_Linux_Chat](https://t.me/Pad_6S_Pro_Linux_Chat) |
| **项目维护者** | 在群内联系管理员或维护者 `@code002-2` |

### 报告内容建议

为帮助我们快速定位和修复，请提供以下信息：

1. **漏洞类型** — 越权、信息泄露、命令注入、缓冲区溢出等
2. **影响范围** — 影响的脚本、配置文件或构建产物
3. **复现步骤** — 从克隆仓库到复现漏洞的完整流程
4. **潜在影响** — 攻击者可利用该漏洞达成什么效果
5. **修复建议** — 如果有初步的修复方案，欢迎一并提供

### 处理流程

1. 我们会在 **48 小时** 内确认收到报告
2. 评估漏洞严重性并制定修复计划
3. 修复完成后发布安全公告
4. 根据贡献者的意愿决定是否在致谢名单中署名

## 安全最佳实践

### 对于贡献者

- 不要在 `.env` 文件或任何脚本中硬编码密钥、Token 或密码
- `sudo` 提权的脚本需谨慎处理用户输入，防止参数注入
- 使用 `shellcheck` 检查 shell 脚本中的潜在安全问题
- 固件和驱动文件尽量从官方源获取，避免供应链攻击

### 对于用户

- 从 [Releases 页面](https://github.com/code002-2/Xiaomi-pad-6s-pro-Linux/releases) 下载预构建镜像，不要使用来源不明的二进制文件
- 首次登录后立即修改默认密码
- 在安全的环境下执行刷机操作

## 范围

本安全策略覆盖仓库中的所有内容，包括但不限于：

- `*.sh` 构建脚本
- `*.config` 内核配置文件
- `firmware/`、`firmware-xiaomi-sheng/` 固件
- CI/CD 工作流定义
- 打包的分发镜像（`*.img`、`*.7z`、`*.deb`）
